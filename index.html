self-introduction

  • html

    시멘틱 태그를 활용하여 콘텐츠의 의미와 구조를 명확히 표현함으로써, 검색 엔진의 콘텐츠 이해와 웹 페이지 사용자의 접근성을 고려하여 작업합니다. 이는 정보 구조화의 중요성을 강조하며, 보조 기술을 활용하는 사용자들에게 웹 페이지를 보다 쉽게 이해할 수 있도록 돕습니다.

  • css

    플렉스와 그리드 레이아웃을 이용하여 웹 페이지를 구성하고, rem, vw, vh와 같은 반응형 단위와 미디어 쿼리를 활용하여 반응형 웹을 개발할 수 있습니다. 이를 통해 다양한 기기와 화면 크기에서 일관된 사용자 경험을 제공함으로써, 사용자들에게 최적화된 웹 환경을 제공하는 것이 가능합니다.

  • scss

    변수를 활용하여 색상, 폰트, 여백 등을 한 곳에 저장하고 필요할 때 재사용할 수 있습니다. 또한 임포트 기능을 이용하여 여러 파일을 하나로 합치고, 중첩 구조로 코드를 깔끔하게 정리할 수 있습니다. 이러한 기능들을 통해 코드 작성이 편리해지며, 유지보수가 용이해집니다. 추가로 반응형 디자인을 위한 기능들을 활용하여 다양한 디바이스와 화면 크기에 맞춘 반응형 웹을 만들어낼 수도 있습니다.

  • design

    인터랙티브 홈페이지는 사용자와의 상호작용을 중요시하며, 동적 요소와 애니메이션을 활용하여 생동감 넘치는 웹 경험을 제공합니다. 이를 통해 특별한 경험과 기억에 남을 사용자 상호작용을 만들어냅니다. 더불어 GSAP를 활용하여 다채로운 애니메이션 효과를 구현하고, jQuery를 활용하여 상호작용이 가능한 요소들을 조작하고 제어할 수 있습니다.

  • 웹접근성

    웹접근성을 고려하여 WAI-ARIA와 IR/IS를 적용했습니다. 마크업 순서와 클래스명을 신중하게 고려하여 작성하여 웹 요소들의 접근성을 높였습니다. 이로써 스크린 리더와 같은 보조 기기를 사용하는 사용자들에게 더 나은 경험을 제공할 수 있었습니다. 이러한 기술과 아키텍처를 적용하여 웹접근성을 향상시키고, 모든 사용자들에게 동등한 웹 경험을 제공하는 데 최선을 다하고 있습니다.

  • 시멘틱 마크업

    웹페이지 구성에 시맨틱 마크업을 주로 활용했습니다. 각 요소에 의미 있는 태그를 사용하여 문서의 내용과 구조를 명확하게 표현했습니다.시맨틱 태그를 적절하게 활용하여 웹사이트를 구획하고 콘텐츠를 논리적으로 그룹화했습니다. 시맨틱 마크업을 통해 웹페이지의 의도와 목적을 명확하게 전달하고, 사용자들에게 더 나은 접근성과 이해도를 제공하기 위해 노력했습니다.
